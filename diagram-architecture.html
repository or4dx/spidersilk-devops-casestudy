CSV Processor — System Architecture

Go · Kubernetes (KOPS) · Nginx sidecar · AWS S3 · Helm · Ansible

Request flow

  • • Browser → Nginx sidecar :80 → Go app :8080
  • • Nginx serves /static/ from emptyDir volume
  • • Go app parses CSV, renders result to browser
  • • Processed file uploaded to S3 via AWS SDK v2
  • • S3 listing drives the previously-processed-files page

Pod design

  • • Nginx + Go app run in the same Kubernetes pod
  • • emptyDir volume shared for static assets (CSS/JS)
  • • Init container (app image) populates volume at start
  • • Nginx serves /static/ — app never handles static files
  • • HPA auto-scales between 2 and 10 replicas at CPU 60%

Config and storage chain

  • • Ansible group_vars: source of truth for all env config
  • • Ansible Vault encrypts secrets at rest in source control
  • • Helm renders values.yaml and deploys to Kubernetes
  • • ConfigMap + Secret deliver config to pod at runtime
  • • S3 lifecycle: Standard → Standard-IA → Glacier IR → Deep Archive
